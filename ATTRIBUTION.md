## Song

**"Let Me Call You Sweetheart"** — words & music by Beth Slater Whitson and
Leo Friedman, published 1910. Both composition and lyrics are in the public
domain in the US.

**Recording used:** *The Jubileers* (vocal quartet & piano), from the series
"Let's All Howl," catalog 1801-A. Digitized 78rpm transfer, hosted on the

https://archive.org/details/78_let-me-call-you-sweetheart_the-jubileers-friedman-whitson_gbia0062289a

`public/audio/let-me-call-you-sweetheart.mp3` is that file, unmodified.

## Lyric timing

Every timestamp in the `LYRICS` array in `src/lyrics-data.mjs` — both each
line's `start`/`end` and every individual word's `s`/`e` inside its `words`
array — was produced by running
[faster-whisper](https://github.com/SYSTRAN/faster-whisper) with word-level
timestamps directly on the recording above:

```python
from faster_whisper import WhisperModel
model = WhisperModel("small", device="cpu", compute_type="int8")
segments, info = model.transcribe(
    "let-me-call-you-sweetheart.mp3", word_timestamps=True, language="en"
)
for seg in segments:
    for w in seg.words:
        print(w.start, w.end, w.word)
```

then checked against the printed 1910 lyric sheet for spelling (Whisper's
transcription is used for **timing**, not wording). One exception: the final
tag line (the last ~1.4s of the recording, a soft fade-out) came back from
Whisper with all ten words collapsed onto the same timestamp — those ten
words were evenly re-interpolated across the segment's real start/end
instead of using the (degenerate) per-word output; every other line's word
timings are Whisper's actual output, unmodified.

## Motion / choreography (Theatre.js)

Every line's, camera's, mood-wash's, and glass hero object's on-screen motion
is a real [Theatre.js](https://www.theatrejs.com/) keyframe sequence,
evaluated by `@theatre/core` at runtime from
`public/assets/theatre-state.json`. That JSON is generated by
`tools/build-theatre-state.mjs` rather than authored by hand in Theatre.js
Studio's visual timeline editor — there's no display to run Studio's UI
against in this project's build environment, and, separately, Theatre.js
0.7.2's public `studio.transaction()` API only writes *static* prop
overrides, not sequenced keyframes, unless a prop has already been marked
"sequenced" by Studio's own right-click UI action (there is presently no
public API for that; see the still-open
[theatre-js/theatre#411](https://github.com/theatre-js/theatre/issues/411)).
Theatre.js does, however, publicly export its on-disk project-state format
as `__UNSTABLE_Project_OnDiskState`, and its own type definitions note that
format "could also be useful for users who manually edit the project state"
— so `tools/build-theatre-state.mjs` constructs that exact JSON shape
directly from the real per-line/per-word timing above. `@theatre/core` reads
and interpolates it completely normally at runtime; only the authoring step
is a script instead of mouse drags. The per-word kinetic entrance and the
sung/unsung sweep stay outside Theatre.js on purpose — they're a discrete
state change at a real timestamp, not a continuously tweened value; see the
doc comment on `main()` in `src/main.js`.

## The three Three.js glass objects (heart / bloom / ribbon)

These are **procedurally authored geometry** (`src/scene/heroObjects.js`),
not a downloaded 3D model. Free CC0 catalogs were investigated first:

- [Poly Pizza](https://poly.pizza) and [Quaternius](https://quaternius.com)
  have CC0 heart/flower models, but the site is a JS-rendered SPA whose
  actual download links aren't reachable from a plain HTTP fetch in this
  environment, and their whole aesthetic (flat-shaded low-poly game props)
  fights the painterly glass look this page wants anyway.
- [Kenney's Nature Kit](https://kenney.nl/assets/nature-kit) *is* a plain,
  directly-downloadable static zip (confirmed working, CC0) — but its
  flower models are flat billboard game sprites, not a fit either.

Rather than force a mismatched free asset, the heart, the abstracted glass
"bloom," and the satin ribbon are all built from scratch with `THREE.Shape` /
`ExtrudeGeometry` (the ribbon uses `extrudePath` along a Catmull-Rom curve)
and rendered with `MeshPhysicalMaterial` (`transmission`, `clearcoat`) lit by
a `RoomEnvironment` PMREM map — a completely license-free, guaranteed-to-load,
guaranteed-on-palette result, and arguably the more advanced Three.js skill
of the two options.

## GIF / PNG assets

The previous build used GIPHY sticker GIFs for its decorative imagery. They
were dropped entirely in this rebuild for two reasons the redesign was
explicitly asked to fix: GIFs are capped at their source frame rate (visibly
stuttery next to Theatre.js's real interpolation), and GIPHY's stickers span
several mismatched illustration styles that fought the page's single
painterly look. Every remaining decorative image is a transparent PNG from
one consistent source (OpenMoji), so — even after individual color
grading — they read as one family rather than a assorted set of clip art.
