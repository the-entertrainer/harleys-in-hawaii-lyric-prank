# My Sweetheart

A single-page lyric experience for a genuinely public-domain 1910 love song
â€” **Dreamy Maximalist Collage**: kinetic Fraunces typography fills the
screen, Three.js bokeh and three procedurally-authored glass objects (a
heart, an abstracted bloom, a satin ribbon) breathe behind it, and every bit
of motion â€” line staging, camera dolly, background mood, glass-object

play button.

## What's actually in this build

- **Real audio, real timing, down to the word.**
  `public/audio/let-me-call-you-sweetheart.mp3` is a 1924 Jubileers
  vocal-quartet recording of the fully public-domain 1910 song "Let Me Call
  You Sweetheart" (Leo Friedman / Beth Slater Whitson). Every lyric and
  every word in `src/lyrics-data.mjs`'s `LYRICS` array carries its own
  start/end timestamp, produced by running
  [faster-whisper](https://github.com/SYSTRAN/faster-whisper) with
  word-level timestamps directly on that recording. See `ATTRIBUTION.md`
  for the full method.
- **Every line, the camera, the background mood, and all three glass
  objects are real Theatre.js sequences.** `@theatre/core` drives
  position/opacity/scale/rotation/blur/font-weight for every lyric line,
  a slow cinematic camera dolly, a hue/warmth crossfade between song
  sections, and the scale/position of the three Three.js hero objects â€”
  all from `public/assets/theatre-state.json`, an on-disk Theatre.js
  project state (the same format Theatre.js Studio itself exports).
  `src/main.js` does nothing more than scrub `sheet.sequence.position` to
  `audio.currentTime` every frame and apply whatever each object's
  `onValuesChange` hands back. See `tools/build-theatre-state.mjs` for why
  that file is generated by a script rather than authored by hand in
  Theatre.js Studio's visual editor (short version: there's no display to
  run Studio's UI against in a headless build environment, and â€” as of
  Theatre.js 0.7 â€” there's no supported public API for creating sequenced
  keyframes from code either).
- **Three procedurally-built glass objects, not a clipart pile.** A heart,
  an abstracted glass "bloom," and a flowing satin ribbon â€” built from
  `THREE.Shape`/`ExtrudeGeometry` and rendered with a physically-based
  transmissive material â€” each own one whole chorus/finale stanza instead
  of a new icon popping in on every line. See `src/scene/heroObjects.js`
  and `ATTRIBUTION.md` for why these are original geometry rather than a
  downloaded model.
- **Sparse, meaningful imagery.** Most of the song is pure typography. The
  handful of 2D image accents that do appear are chosen because the
  picture *is* the lyric (the sun for "keep the love light glowing," a
  crescent moon handing off into the moonlit verse) â€” see `HERO_WINDOWS`,
  `COLLAGE_ASSETS` and `GAP_ASSETS` in `src/lyrics-data.mjs`.
- **Word-by-word kinetic entrance + sung/unsung highlighting**, both driven
  directly off each word's real timestamp: a 3D rotate-and-blur-in reveal
  when a word's about to be sung, then a progress-wipe underline exactly
  when it is. Both stay outside Theatre.js on purpose â€” discrete
  timestamp-triggered state, not a continuously tweened value (see the
  doc comment on `main()` in `src/main.js`).
- **A randomized pink accent.** A curated set of emotionally-loaded words
  (sweetheart, love, true, dreaming, moonlight, whisperâ€¦) has a high but not
  guaranteed chance of turning pink instead of plain ink, re-rolled on every
  page load.
- **A real ending, not a silent reset.** When the song actually finishes, a
  glass heart pulses, a scattering of the remaining illustrations drift in,
  and "From Naveen, with love" fades up, held until you tap play again.
- **Mobile-aware throughout**: capped device-pixel-ratio and reduced
  particle/geometry complexity on small screens, a responsive Fraunces
  optical-size axis, safe-area insets, and a `prefers-reduced-motion` path
  that freezes the decorative motion in both the DOM and the WebGL layer.

  build-theatre-state.mjs        regenerates public/assets/theatre-state.json from src/lyrics-data.mjs

## Run locally

```bash
npm install
npm run dev       # Vite dev server with HMR
```

Or to check the exact production build:

```bash
npm run build
npm run preview   # serves dist/ at http://localhost:8080
```

Vite's dev/preview servers both honor HTTP Range requests, which audio
seeking depends on.

## Changing the choreography, timing, or hero objects

1. Edit `src/lyrics-data.mjs` (timing, words, section labels, which asset or
   hero window goes with which line â€” see `HERO_WINDOWS`, `COLLAGE_ASSETS`,
   `GAP_ASSETS`, `MOODS`).
2. Edit the motion recipes in `tools/build-theatre-state.mjs` if you want to
   change the per-line variants, the camera dolly, or the hero-object
   entrance/exit tracks. Edit `src/scene/heroObjects.js` to change the
   glass geometry itself.
3. Run `npm run build:theatre` to regenerate `public/assets/theatre-state.json`.
4. `npm run smoke` samples a few values from the regenerated state (line
   motion, camera, hero scale) to sanity check it before you reload the
   page. `npm run smoke:page` drives the real page end to end in headless
   Chromium and screenshots a few timestamps â€” point `SMOKE_URL` at your
   dev server (and set `SMOKE_MOBILE=1` for a phone-sized viewport).

## Deploy

**Vercel:** import this GitHub repo at vercel.com/new. It auto-detects Vite
