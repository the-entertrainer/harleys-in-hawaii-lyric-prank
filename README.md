# My Sweetheart

A single-page lyric experience for a genuinely public-domain 1910 love song
— staged as if it's playing on a **centered 3D vintage TV**. Real word-level
audio timestamps drive a broadcast on the TV's CRT screen: fat retro display
type stacking word-by-word exactly as it's sung, a photographic anchor per
song section, the key word writing itself in connected script, all wrapped
in real vintage scanline/static/glitch noise — while the model's own PBR
material keeps a genuine specular highlight glinting on the glass. There is
exactly one control: a play button.

## What's actually in this build

- **Real audio, real timing, down to the word.**
  `public/audio/let-me-call-you-sweetheart.mp3` is a 1924 Jubileers
  vocal-quartet recording of the fully public-domain 1910 song "Let Me Call
  You Sweetheart" (Leo Friedman / Beth Slater Whitson). Every lyric and
  every word in `src/lyrics-data.mjs`'s `LYRICS` array carries its own
  start/end timestamp, produced by running
  [faster-whisper](https://github.com/SYSTRAN/faster-whisper) with
  word-level timestamps directly on that recording. See `ATTRIBUTION.md`.
- **A real 3D TV, not a flat backdrop.** A licensed glTF model
  (`public/models/retro-tv/`, CC BY-NC 4.0 — see `ATTRIBUTION.md`) rendered
  with Three.js (`src/scene/tvScene.js`): PBR materials, a PMREM environment
  for real reflections, and a camera that fits the model to any aspect ratio.
  The CRT screen's own baseColor texture is replaced at runtime with a live
  `CanvasTexture` — everything else on the model (wood bezel, knobs, decals)
  stays exactly as authored, and the metallicRoughness/normal maps are never
  touched, so the glass keeps its real specular highlight.
- **Broadcast scenes, not a lyric scroller.** `SCENES` in
  `src/lyrics-data.mjs` divides the song at its section boundaries.
  `src/scene/tvScreen.js` draws each scene as a lower-third title card onto
  the screen canvas — headline words popping in typewriter-style at their
  real sung timestamp, the key word ("sweetheart," or the scene's emotional
  word) inking itself on in connected script, one photographic anchor
  behind it — with scanlines, animated static, chromatic aberration, and
  occasional roll glitches layered on top for a real CRT look.
- **Every tweened value is a real Theatre.js sequence.** Scene crossfades,
  each anchor's rise-in and idle float, the gradient's hue/warmth shift
  between scenes — all keyframed tracks in `public/assets/theatre-state.json`,
  generated by `tools/build-theatre-state.mjs` (see that file for why it's
  generated rather than hand-authored in Theatre.js Studio) and evaluated by
  `@theatre/core` scrubbed to `audio.currentTime`. Word reveal stays
  deliberately outside Theatre.js: a word appearing at its sung timestamp is
  discrete state, not an animated value.
- **Real photographic cutouts.** Every anchor is a licensed Adobe Stock
  photograph with its background removed (Photoshop API) — a genuine
  gramophone, rotary phone, dove, candlestick, pearl necklace, typewriter,
  bouquet, ring, vinyl disc, pocket watch. One per scene, drawn onto the TV
  screen behind that scene's text. See `ATTRIBUTION.md` for every source.
- **Typography matched to the reference**: Titan One (fat retro display),
  Cookie (connected script), Oswald (caption) — self-hosted woff2, no
  external font dependency at runtime, drawn with `CanvasRenderingContext2D`
  onto the screen texture.
- **A real ending.** When the song finishes, the TV's last broadcast is a
  closing card — "Made with love by Naveen" — and the screen stays on,
  idling with scanline shimmer instead of cutting to a separate overlay.
- **Mobile-first by construction**: capped device-pixel-ratio, disabled
  antialiasing on small screens, `prefers-reduced-motion` disables camera
  parallax and CRT roll/flicker (static scanlines/vignette/grain stay, since
  those aren't motion), safe-area insets throughout.

## Repo layout

```
index.html                     Vite entry (HTML shell only — see src/main.js)
src/
  main.js                       audio + Theatre.js wiring, parallax, the merged render loop
  style.css                     page-level chrome (ambient gradient, vignette, play button)
  lyrics-data.mjs                word timing + SCENES broadcast definitions (single source of truth)
  scene/
    tvScene.js                   Three.js renderer/camera/lighting/GLTFLoader, screen-texture atlas patching
    tvScreen.js                  the CRT screen's own canvas — layout, word reveal, vintage effects
public/
  models/retro-tv/                licensed glTF TV model + textures (see ATTRIBUTION.md)
  assets/theatre-state.json      generated Theatre.js sequence (regenerate, don't hand-edit)
  assets/photo/                  licensed, background-removed photo cutouts (see ATTRIBUTION.md)
  fonts/                         self-hosted Titan One / Cookie / Oswald (woff2)
  audio/                         the mp3 + its own README
tools/
  build-theatre-state.mjs        regenerates public/assets/theatre-state.json from src/lyrics-data.mjs
  smoke-test.mjs                 loads the generated state into real @theatre/core and samples values
  smoke-test-page.mjs            drives the real page in headless Chromium, screenshots each scene
```

## Run locally

```bash
npm install
npm run dev       # Vite dev server with HMR
```

Or the production build:

```bash
npm run build
npm run preview   # serves dist/ at http://localhost:8080
```

## Changing the broadcast, timing, or imagery

1. Edit `src/lyrics-data.mjs` — `SCENES` controls each broadcast's
   boundary, anchor photo, gradient hue, and which word ranges land in the
   headline / script / sub / spoken slots.
2. Edit `tools/build-theatre-state.mjs` to change the build-in/float/fade
   choreography itself.
3. Edit `src/scene/tvScreen.js` to change the screen's layout or vintage
   effects (scanlines, static, chromatic aberration, glitch).
4. `npm run build:theatre` to regenerate the sequence, `npm run smoke` to
   sanity-check sampled values, and `npm run smoke:page` (with `npm run
   dev` running; `SMOKE_MOBILE=1` for a phone viewport) to screenshot every
   scene.

## Deploy

**Vercel:** auto-detected Vite (`npm run build`, output `dist/`) — zero
config. **GitHub Pages:** publish `dist/`.
